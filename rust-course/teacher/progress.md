## 易错点

- 容易把 `cargo clippy` 误认为"语法检查"；它更偏向代码异味、潜在 bug、非 idiomatic 写法
- 容易把遮蔽和 `mut` 混为一谈；遮蔽是新绑定，`mut` 是原绑定可变
- 在 Windows 上即使装了 Rust，也可能因 `link.exe` 缺失导致第一次构建失败，需要正确安装 MSVC Build Tools/SDK
- 容易把 `"A"` 和 `'A'` 混为一谈；前者是字符串切片 `&str`，后者才是字符 `char`
- 容易把"有无分号"和"语句/表达式/返回值"关系说反；要反复练到顺口
- 容易把 `b'A'` 和 `'A'` 混为一谈；前者是单字节 `u8`，后者是 `char`
- 容易把浮点数当成精确十进制来比较；要优先建立"近似值"心智模型
- 容易在位移时忽略位数上限；`u8 << 8`、`u8 >> 8` 都会溢出报错
- 容易把 `'A'`、`"A"`、`b'A'` 混在一起；三者类型完全不同：`char`、`&str`、`u8`
- 容易忘记 `()` 只能用 `{:?}` 打印，不能用 `{}`
- 容易在表达式末尾多加分号，导致函数返回 `()` 而非期望类型，触发 `E0308`
- 容易把 `return` 和末尾表达式混用场景搞反；`return` 用于提前退出，末尾表达式用于正常返回
- 容易把 `-> !` 理解为"报错"而不是"永不返回"；`!` 不等于 panic，无限循环也是 `!`
- **容易忘记函数传参也会 Move：`takes_ownership(s)` 后 `s` 就失效了**
- **容易在需要借用时忘记加 `&`：应该写 `&s` 而不是 `s`**
- **容易混淆 Move 和 Copy：忘记检查类型是否实现了 `Copy` trait**
- **容易滥用 `clone()`：忘记了它有性能代价，该用借用时却用 clone**
- **容易忘记 `&` 只是借用，修改需要 `&mut`**
- **容易试图创建多个可变引用：忘记"同一作用域只能有一个 `&mut`"规则**
- **容易混用可变和不可变引用：忘记两者不能同时存在**
- **容易误解引用作用域：以为要到花括号结束才释放（NLL 已优化）**
- **容易返回局部变量的引用：忘记悬垂引用问题，应该返回所有权**
- **容易把 `String` 和 `&str` 混成“都是字符串”；要先分清是否拥有所有权**
- **容易误以为字符串切片按字符索引；其实按字节索引，中文场景最容易踩坑**
- **容易把 `replace()` 记成“原地修改”；它返回新字符串，不修改原值**
- **容易忘记 `pop()` 的返回值是 `Option<char>` 而不是 `char`**
- **容易把 `remove(index)` 理解成“取子串”；其实它删除并返回一个字符**
- **容易忽略 `+` 会 move 左侧 `String`；拼接多段字符串时更适合考虑 `format!`**
- **容易把 `chars()` 和 `bytes()` 混淆；前者产出 `char`，后者产出 `u8`**
- **容易把元组索引写错位；第二个元素是 `.1`，不是 `.2`**
- **容易把 `_` 误解成“匿名变量”；它更准确是“忽略该位置，不绑定名字”**
- **容易在含 `String` 的元组上忽略 move：字段被取出或解构后，原元组往往不能再整体使用**
- **容易把“元组能装多个值”误以为“适合所有场景”；一旦字段语义明确，结构体通常更清晰**
- **容易误以为可以只让结构体某个字段可变；实际上要修改字段，整个实例绑定必须是 `mut`**
- **容易把结构体字段初始化写成 `=`；正确语法是 `field: value`**
- **容易在没有同名变量时误用字段简写；`title` 简写的前提是当前作用域里真的有名为 `title` 的变量**
- **容易把 `..old` 理解成“整份复制”；其实只有 `Copy` 字段是拷贝，`String` 等字段会 move**
- **容易把“字段被 move”说成“立刻 drop”；更准确是所有权已转移，原结构体因此不再完整可用**
- **容易混淆元组结构体和单元结构体：前者有字段但无字段名，后者根本没有字段**
- **容易把“结构体字段也能放复杂类型”误说成“结构体只能放基本类型”；实际上结构体字段几乎可以是任何类型**
- **容易把变量的类型和变体携带的数据形状混淆：`Message::Write(String)` 这个值的整体类型是 `Message`，不是 `String`**
- **容易只记住“枚举代码更短”，却没抓住其真正优势是：把“分支”与“该分支对应的数据形状”绑定进同一个类型**
- **容易把 `Option<T>` 理解成“语法版判空”，而忽略它本质上是一个必须显式处理分支的枚举类型**
